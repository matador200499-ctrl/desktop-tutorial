# أتمتة قناة اللقطة (Al Laqta) — نظام كامل مجاني

نظام أتمتة كامل بينشر فيديوهات على يوتيوب **4 مرات يوميًا** من غير أي تدخل بشري بعد الإعداد الأولي، وكل مكوناته مجانية:

| المرحلة | الأداة | التكلفة |
|---|---|---|
| كتابة السكريبت (مقسّم لمشاهد) | Claude API | قروش قليلة/فيديو (عندك المفتاح) |
| التعليق الصوتي | edge-tts (أصوات Microsoft Edge العصبية) | مجاني 100% |
| مقاطع الفيديو | Pexels (فيديوهات ستوك حقيقية بدون حقوق) | مجاني 100% |
| تركيب الفيديو + الكتابة على الشاشة | FFmpeg | مجاني 100% |
| الرفع على يوتيوب | YouTube Data API | مجاني 100% |
| الجدولة والتشغيل | GitHub Actions | مجاني (بشرط الريبو Public) |

---

## ⚠️ 3 حقائق مهمة قبل ما تبدأ

1. **edge-tts خدمة غير رسمية** من مايكروسوفت (مش API معلن رسميًا للمطورين)، شغالة ومجانية دلوقتي وبيستخدمها آلاف المطورين، لكن نظريًا ممكن تتغير أو تتوقف من غير سابق إنذار. لو حصل كده، هنحتاج نبدّلها ببديل (زي Google Cloud TTS اللي فيه رصيد مجاني شهري محدود).
2. **الريبو لازم يكون Public** عشان تستفيد من دقائق GitHub Actions المجانية بلا حدود. الريبوهات الـ Private ليها حد 2000 دقيقة/شهر مجانًا، وممكن تخلص بسرعة مع فيديو كل 6 ساعات. متقلقش على أمان مفاتيحك — الـ Secrets بتفضل مخفية حتى لو الريبو Public.
3. **محتوى مولّد بالكامل بالذكاء الاصطناعي وبدون مراجعة بشرية بيحمل مخاطرة سياسات يوتيوب**: يوتيوب بدأ يشدد على المحتوى "المكرر أو المصنّع بكميات كبيرة بدون قيمة مضافة" فيما يخص الأهلية للربح (Monetization). يُنصح تراجع فيديو من كل دفعة بشكل دوري للتأكد من الجودة، ولو حابب تفعّل شارة "محتوى مولّد بالذكاء الاصطناعي" في إعدادات يوتيوب (مطلوبة بحسب سياسة يوتيوب للمحتوى الواقعي المصطنع).

## 1) هيكل المشروع

```
al-laqta-automation/
├── .github/workflows/publish_video.yml   ← الأتمتة الكاملة (مجدولة 4 مرات/يوم)
├── scripts/
│   ├── generate_script.py    ← Claude يكتب السكريبت مقسّم لمشاهد
│   ├── generate_voice.py     ← edge-tts يحوّل كل مشهد لصوت
│   ├── fetch_clips.py        ← يجيب مقطع ستوك من Pexels لكل مشهد
│   ├── assemble_video.py     ← يركب كل حاجة بـ FFmpeg (فيديو + صوت + كتابة)
│   └── upload_youtube.py     ← يرفع الفيديو النهائي على يوتيوب
├── pick_topic.py              ← يختار الموضوع التالي من topics.txt أوتوماتيك
├── topics.txt                 ← قائمة انتظار المواضيع (موضوع في كل سطر)
├── used_topics.txt            ← أرشيف المواضيع المستخدمة (بيتحدث أوتوماتيك)
├── get_refresh_token.py       ← تشغّله مرة واحدة بس على جهازك
├── requirements.txt
└── .gitignore
```

## 2) الإعداد لأول مرة

### أ) اعمل الريبو
اعمل ريبو جديد على GitHub، واختاره **Public** (السبب في الملاحظة رقم 2 فوق). ارفع كل ملفات المشروع.

### ب) مفاتيح YouTube (زي ما اتفقنا قبل كده)
1. [Google Cloud Console](https://console.cloud.google.com/) → مشروع جديد → فعّل **YouTube Data API v3**.
2. OAuth consent screen → External → ضيف حسابك كـ Test User.
3. Credentials → Create OAuth client ID → نوعه **Desktop app** → نزّل `client_secrets.json`.
4. على جهازك: `pip install -r requirements.txt` ثم `python get_refresh_token.py` — هتاخد `client_id` و`client_secret` و`refresh_token`.

### ج) مفتاح Pexels (مجاني)
1. سجّل في [pexels.com/api](https://www.pexels.com/api/) بإيميلك (تسجيل فوري، بدون بطاقة ائتمان).
2. هتاخد API key فورًا من الداشبورد.
3. الحد المجاني: 200 طلب/ساعة و20,000 طلب/شهر — كافي جدًا لـ 4 فيديوهات يوميًا (كل فيديو بيستخدم ~10-12 طلب بحث).

### د) ضيف الـ Secrets في GitHub
Settings → Secrets and variables → Actions → New repository secret:

| اسم الـ Secret | القيمة |
|---|---|
| `ANTHROPIC_API_KEY` | مفتاح Claude بتاعك |
| `PEXELS_API_KEY` | من خطوة (ج) |
| `YT_CLIENT_ID` | من client_secrets.json |
| `YT_CLIENT_SECRET` | من client_secrets.json |
| `YT_REFRESH_TOKEN` | من خطوة (ب-4) |

### هـ) جهّز قائمة المواضيع
افتح `topics.txt` وضيف مواضيع (موضوع في كل سطر). النظام بياخد أول سطر في كل مرة يشتغل، فلازم تتابع وتضيف مواضيع جديدة بشكل دوري (هيظهرلك تحذير في الـ Actions log لما يبقى باقي 4 مواضيع بس).

## 3) بعد كده، مفيش حاجة تعملها

بمجرد ما الإعداد يخلص، الـ workflow هيشتغل لوحده **4 مرات يوميًا** (الساعات مضبوطة في `publish_video.yml`، عدّلها لو حابب توقيت مختلف). كل مرة:

1. ياخد أول موضوع من `topics.txt`
2. Claude يكتب سكريبت مقسّم لمشاهد
3. edge-tts يولّد الصوت
4. Pexels يديله مقاطع فيديو لكل مشهد
5. FFmpeg يركب كل حاجة مع الكتابة على الشاشة
6. يرفع الفيديو على يوتيوب مباشرة كـ Public

تقدر تتابع كل تشغيلة من تبويب **Actions**، وهتلاقي الفيديو النهائي والسكريبت محفوظين كـ Artifact في نتيجة كل Run (لمدة 7 أيام) لو حبيت تراجعهم.

---

## 4) تشغيل يدوي لموضوع معين (اختياري)

من تبويب **Actions** → **Publish Video (Automated)** → **Run workflow** → اكتب موضوع في خانة `manual_topic` لو عايز فيديو عن حاجة محددة خارج الطابور، من غير ما ياخد من `topics.txt`.

---

## 5) لو حبيت تعدّل لاحقًا

- **تغيير صوت الراوي**: غيّر `TTS_VOICE` في `scripts/generate_voice.py` (أصوات عربية تانية متاحة: `ar-EG-SalmaNeural`, `ar-SA-HamedNeural`, `ar-SA-ZariyahNeural`).
- **تغيير عدد مرات النشر**: عدّل سطر `cron` في `publish_video.yml`.
- **تغيير مدة الفيديو**: عدّل عدد الكلمات المطلوب في `scripts/generate_script.py`.
